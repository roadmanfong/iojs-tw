io.js 週報 2015.03.13

io.js 釋出 1.5.1 版

@rvagg 在 3/9 (一) 釋出了 io.js v1.5.1。可以在 GitHub 上 找到完整的變更日誌。

主要變更

  • tls: 之前回報的 TLS 記憶體洩漏已經在這次釋出中藉由多個 commit 解決了。現在的測試指出，可能 還有一些洩漏問題。完整的進度追蹤在 #1075。
  • http: 修復一個回報在 joyent/node#9348 和 npm/npm#7349 的錯誤。當 'error' 事件發生時，待處理 (pending) 的資料沒有被完整讀出，導致 socket.destroy() 的一個 assert 失敗。 (Fedor Indutny) #1103

已知問題

  • 可能仍然有與 TLS 相關的記憶體洩漏，細節請參閱 #1075。
  • Windows 仍舊有使用者回報存在一些測試錯誤的情形，我們正優先追查這些問題的原因。請參閱 #1005.
  • 在 REPL 中代理對 (Surrogate pair) 會凍結終端 #690
  • 不能編譯 io.js 為靜態函式庫 #686
  • 1.0.2 重新加入的 process.send() 不如文件上所說為同步地，參閱 #760 和 #774 的修補
  • 當 DNS 查詢進行到一半時呼叫 dns.setServers()，會導致程序當機並 assert 失敗 #894

io.js 週報 2015.02.20

io.js 釋出 1.3.0

值得注意的改變包括：

  • url: url.resolve('/path/to/file', '.') 現在會回傳包括尾端斜線的 /path/to/，url.resolve('/', '.') 則回傳 / #278 (Amir Saboury)
  • tls: tls 和 https 預設加密套件改用 PFS (Perfect Forward Secrecy，一種兼容於主流瀏覽器加密套件) 。此外，不安全的 RC4 加密已經被移除。如果你真的需要 RC4，請指定自己的加密套件。 #826 (Roman Reiss)

社群中的重要事件

  • Node 管理 - William Bert 建立 http://nodegovernance.io/ 來警告 Joyent 的執行長 Scott Hammond，社群希望 Node 基金會的技術委員會能基於 io.js 的開放式治理模式。社群的回應真是 太棒了！
  • Node.js 和 io.js 效能提高 - Raygun.io 最近對 Node.js 和 io.js 兩者都做了效能測試，並且都藉由釋出提高了效能！閱讀整篇文章.
  • LTTng 基礎 - 使用者 jgalar 在 asciinema 上建立使用 io.js 的 LTTing 基礎
  • io.js 規劃路線投影片 - 展示 io.js 當前規劃路線的投影片。

已經支援 io.js 的

  • TravisCI 新增 io.js。每週更新張貼的最後一天，Hiro Asari (あさり) 推了一則 tweet 說 10% 的 Node 專案已經運行 io.js。
  • @thlorenz 更新 nad，Node Addon 開發者，以 支援 io.js。
  • Catberry.js 增加 io.js 的支援。
  • mongodb 的官方 node 模組在 v. 2.0.16 2015-02-16 支援 io.js。
  • The Native Web 現在擁有 io.js Docker 容器
  • okTurtles 的 DNSChain 增加對 io.js 的支援。
  • actionHero 的 TDPAHACLPlugin 和 TDPAHAuthPlugin 現在支援 io.js。
  • node-sass 在 node-sass v. 2.0.1 增加對 io.js 1.2 的支援。
  • total.js 在 v. 1.7.1 增加對 io.js 的支援。
  • Clever Cloud 已經增加 io.js 的支援
